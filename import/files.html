Introduction

Importing data from files

In order to load the database with data from archived exports from another database, there are several scripts that will read .xlsx spreadsheet files and EZproxy .log files and insert the appropriate records into the database.

  • patrondata_ingest.py
  • instruction_ingest.py
  • ezproxy_ingest.py
  • patroncounter_ingest.py

All four of these scripts require that you install Python on your computer in order to run them. The instructions for installing Python and the neccessary Python add-ons are included in Appendix A.

The scripts access the Access database through an ODBC connection, so you must also go through the steps in the section on Setting up your database before running them.

Once those steps are complete, you can run one of the scripts by simply double-clicking on it. It should open up a window that will print messages to the screen letting you know how the import process is going. If it immediately exits, it means that some part of the setup for your Python environment or your ODBC connection wasn't correct.

Import order

Parts of the import process depend on each other, so data should be imported in a specific order.

  1. Import PatronData using patrondata_ingest.py
  2. Import Instruction using instruction_ingest.py
  3. Import EZproxy data using ezproxy_ingest.py
  4. Import PatronCounter data using patroncounter_ingest.py

Note that Microsoft Access is limited in the amount of data it can store. PatronCounter data in particular can grow rapidly, so try importing just a few months of data at a time.
